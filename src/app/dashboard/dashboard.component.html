Welcome / Review

This application was created by Chase Richardson to showcase the creation of an angular app in response to a challenge from Prime Healthcare Services. Here are the requirements.

Prime Assessment:

  1. Create an Angular (version 4) app that consumes some type of public JSON web service and uses the results - your choice on which web service and which control.

    • Application was built using Angular 4.0.3.
    • The app mimics a very basic level of employee management for a hospital, and the employee module consumes data from these two sources:
      • http://demo1555654.mockable.io/getemployees

  2. Demonstrates understanding of module and component architecture

    • Application is divided into the following structure:
    • app (routing, app component, app module)
      • dashboard (component, html, css)
      • employees
        • employee-list (component, html, css)
        • employee (component, html, css)
        • shared (service & model)

  3. Demonstrate use of RxJS

    • Created observable stream based on click event of a button, and chained the buffer, map, filter operators to demonstrate it's value.

  4. Include at least one unit test

    • Unit test is installed at very for the root level AppComponent.

  5. Use best practices - clean code, comments, error handling, etc

    • Best practices and architecture followed according to: Angular Docs. I originally started with the Quick Start Guide, and as I made changes to create my own application, I followed architecture guidelines there defined.

  6. Track how long it takes you to complete this task

    • [6] Monday: 7:30am - 11:30am; 2:30pm - 4:30pm; (Lessons in Angular4/Typescript, Initial Scaffolding)
    • [] Wednesday: 9:00am - 1:00pm; 2:00pm - 7:30pm; (Dashboard, EmployeeList, Http.Get, and RxJS implementation, initial push)

  7. Put the project on Github and send it back for review

    • https://github.com/epitrochoider/prime-healthcare-challenge

  8. Utilizes component routing

    • Using routing for navigation of application

  9. Demonstrate data binding

    • Data binding implemented in EmployeeListComponent and EmployeeDetailComponent

  10. Demonstrate passing data between components with Inputs

    • @Inputs passing of data in Clicker module: the counter indicator is passed through a child component.

  11. Demonstrate a basic understanding of HTML/CSS for UI layout and styling

    • Implemneted bootstrap framework and an basic admin theme for a straightforward interface.
